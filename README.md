# Biophotonica

Приложение на Python для генерации случайных начальных координат фотонов
в осесимметричном пучке с радиальным профилем интенсивности

\[
p(r) = \exp\left(-\left(\frac{r}{\sqrt{2a}}\right)^2\right), \quad r \in [0, \infty)
\]

## Что делает программа

На вход подаются:

- количество фотонов `N`
- параметр пучка `a`

На выходе программа формирует:

- `N` случайных положений фотонов в плоскости `XY`
- `CSV`-файл с координатами `x, y, r, theta`
- итоговую фигуру с четырьмя графиками:
  - распределение фотонов в плоскости `XY`
  - радиальное распределение `r` и аналитическая кривая
  - одномерное распределение `x` и аналитическая кривая
  - осевое сечение двумерной плотности `rho(x, 0)` и аналитическая кривая

Такой набор графиков закрывает неоднозначность формулировки задания: можно показать
и радиальный профиль, и осевое сечение, и распределение координат.

## Математическая модель

После нормировки в плоскости `XY` двумерная плотность имеет вид:

\[
\rho(x, y) = \frac{1}{2\pi a}\exp\left(-\frac{x^2 + y^2}{2a}\right)
\]

Тогда радиальная плотность:

\[
f(r) = \frac{r}{a}\exp\left(-\frac{r^2}{2a}\right), \quad r \ge 0
\]

Это распределение Рэлея, поэтому выборка генерируется по формулам:

- `r = sqrt(-2 a ln(1-u))`, где `u ~ U(0, 1)`
- `theta ~ U(0, 2pi)`
- `x = r cos(theta)`
- `y = r sin(theta)`

## Запуск

```bash
python photon_beam.py --N 10000 --a 4.0 --seed 42
```

После запуска будут созданы:

- `photon_positions.csv`
- `photon_beam.png`

Также в консоль выводятся несколько численных проверок, чтобы было проще убедиться,
что моделирование соответствует теории.

## Автоподбор N и bins

Можно включить автоматический подбор параметров моделирования по порогу ошибки:

```bash
python photon_beam.py --a 4.0 --N 1000 --auto-tune --tolerance 0.05 --target x_marginal --trials 5 --seed 42
```

Доступные цели сравнения:

- `x_marginal` - одномерное распределение по координате `x`
- `r_profile` - радиальное распределение `r`
- `axis_section` - осевое сечение двумерной плотности `rho(x, 0)`

После автоподбора программа выбирает найденные `N` и `bins`, затем строит итоговые
координаты и графики уже с этими параметрами.

В устойчивом режиме ошибка усредняется по нескольким последовательным `seed`, начиная
с указанного значения `--seed`. Это уменьшает вероятность того, что параметры будут
подобраны по случайно слишком удачной одной реализации.

По умолчанию критерий строгий: требуется, чтобы ниже порога были одновременно
`mean_error` и `max_error`. Если нужен более мягкий режим только по средней ошибке,
можно добавить флаг `--mean-only`.
